在线咨询聊天需求整理

参考资料

  • docs/chat/mult_role_views.png：在线咨询 5 种视角示意
  • docs/chat/normal.png：医生端聊天窗口位置示意

角色与关系

  • 患者/家属：UserType.PATIENT，患者档案 PatientProfile，家属关系 PatientRelation
  • 医院主任：DoctorProfile，每位主任对应一个工作室 DoctorStudio（一一对应）
  • 平台医生：DoctorProfile（非主任）
  • 医生助理：AssistantProfile，与医生多对多（DoctorAssistantMap），与医生共同构成工作室成员
  • CRC：SalesProfile（user_type=SALES）

说明：

  • 工作室成员（StudioMember）不建表，按“医生 + 其助理”推导
  • 平台医生与 CRC 可服务多个主任工作室（由绑定关系决定）

会话类型

  • 患者会话（PATIENT_STUDIO）：患者/家属 ↔ 主任工作室（患者侧仅显示工作室名）
  • 内部会话（INTERNAL）：平台医生 ↔ 主任（仅医生端可见）

患者端需求

  1. 患者看到的医生端信息统一显示为“主任工作室”，不暴露真实角色
  2. 同一患者同一时间只归属一个主任工作室
  3. 患者转移工作室后，聊天记录完整迁移，且历史消息保留原工作室名称
  4. 亲情账号权限与患者一致，但医生端显示“姓名（xx家属）”
  5. 附件仅支持图片
  6. 消息同步采用轮询，建议 5 秒一次

医生端需求

  1. 一位主任只有一个工作室；一个工作室可接待多个患者
  2. 平台医生可与患者聊天，也可与主任进行内部会话；患者侧仍显示主任工作室
  3. 平台医生无法解答/需升级时，将问题转发至主任的内部会话
  4. 主任可查看患者会话记录，但不能直接参与与患者的对话
  5. 主任收到平台医生转发时有未读红点提醒；主任通过内部会话回复指示，由平台医生转达患者

会话唯一性

  • 每位患者固定一个患者会话（PATIENT_STUDIO）
  • 内部会话按患者维度固定一条（用于升级沟通）

未决事项

  • 患者转移后旧工作室是否保留只读权限（待定）

开发方案（架构层面）

目标

在现有 chat app 基础上补齐后端模型与接口，前端完成患者端与医生端的统一聊天体验，满足“多角色 + 两类会话 + 升级转发 + 未读提醒 + 轮询同步”。

模块划分

  • chat app：领域模型、业务服务、权限校验
  • web_patient：患者端页面与视图调用（轮询拉取/发送/已读/上传）
  • web_doctor：医生端聊天面板与视图调用（患者列表、未读、会话/发送/升级）
  • users：复用现有 DoctorProfile/AssistantProfile/SalesProfile/DoctorStudio 关系

数据模型（建议）

  • Conversation
    • type：PATIENT_STUDIO / INTERNAL
    • patient：FK PatientProfile
    • studio：FK DoctorStudio（患者会话当前归属工作室）
    • created_by：FK CustomUser
    • last_message_at
    • 约束：(patient, type) 唯一
  • Message
    • conversation：FK
    • sender：FK CustomUser
    • sender_role_snapshot：角色快照（主任/平台医生/CRC/家属）
    • sender_display_name_snapshot
    • studio_name_snapshot：用于转移后历史保留
    • content_type：text / image
    • content：文本或图片地址
    • created_at
  • ConversationReadState
    • conversation + user + last_read_message_id
    • 用于未读统计与红点
  • PatientStudioAssignment
    • patient + studio + start_at + end_at + reason
    • 转移时更新，支持历史追溯

调用方式与同步（建议）

  • 不采用独立 webservice 层，View → Form → Service
  • 轮询策略：5 秒一次，按 last_message_id 增量拉取，空响应返回空列表
  • View 层只做参数接收与校验，业务处理全部在 Service 层完成

Service 层方法清单（原子/可复用）

注释规范遵循 development-guidelines.md（【功能说明】/【使用方法】/【参数说明】/【返回值说明】），以下为建议方法清单。

患者归属（建议放到 PatientService）

  • get_active_studio_assignment(patient)
    • 【功能说明】获取患者当前归属的工作室。
    • 【使用方法】get_active_studio_assignment(patient)
    • 【参数说明】patient: PatientProfile
    • 【返回值说明】PatientStudioAssignment | None

会话

  • get_or_create_patient_conversation(patient, studio=None, operator=None)
    • 【功能说明】获取或创建患者会话（PATIENT_STUDIO），保证唯一性。
    • 【使用方法】get_or_create_patient_conversation(patient, studio, operator)
    • 【参数说明】patient: PatientProfile, studio: DoctorStudio | None, operator: CustomUser | None
    • 【返回值说明】Conversation
  • get_or_create_internal_conversation(patient, studio, operator=None)
    • 【功能说明】获取或创建内部会话（INTERNAL），按患者维度唯一。
    • 【使用方法】get_or_create_internal_conversation(patient, studio, operator)
    • 【参数说明】patient: PatientProfile, studio: DoctorStudio, operator: CustomUser | None
    • 【返回值说明】Conversation
  • list_conversation_messages(conversation, after_id=None, before_id=None, limit=50)
    • 【功能说明】会话消息增量读取，支持轮询与历史加载。
    • 【使用方法】list_conversation_messages(conversation, after_id=123)
    • 【参数说明】conversation: Conversation, after_id/before_id: int | None, limit: int
    • 【返回值说明】List[Message]

消息

  • create_text_message(conversation, sender, content)
    • 【功能说明】创建文本消息，生成展示快照（角色、名称、工作室名）。
    • 【使用方法】create_text_message(conversation, sender, content)
    • 【参数说明】conversation: Conversation, sender: CustomUser, content: str
    • 【返回值说明】Message
  • create_image_message(conversation, sender, image_file)
    • 【功能说明】创建图片消息，校验格式后保存文件并生成消息记录。
    • 【使用方法】create_image_message(conversation, sender, image_file)
    • 【参数说明】conversation: Conversation, sender: CustomUser, image_file: UploadedFile
    • 【返回值说明】Message
  • forward_to_director(patient_conversation, source_message_id, operator, note=None)
    • 【功能说明】将患者消息升级转发至内部会话，并触发未读红点。
    • 【使用方法】forward_to_director(patient_conversation, source_message_id, operator)
    • 【参数说明】patient_conversation: Conversation, source_message_id: int, operator: CustomUser, note: str | None
    • 【返回值说明】Message

已读与未读

  • mark_conversation_read(conversation, user, last_message_id=None)
    • 【功能说明】更新用户在会话内的已读水位。
    • 【使用方法】mark_conversation_read(conversation, user, last_message_id)
    • 【参数说明】conversation: Conversation, user: CustomUser, last_message_id: int | None
    • 【返回值说明】ConversationReadState
  • get_unread_count(conversation, user)
    • 【功能说明】计算单会话未读数。
    • 【使用方法】get_unread_count(conversation, user)
    • 【参数说明】conversation: Conversation, user: CustomUser
    • 【返回值说明】int
  • get_unread_counts(user, conversation_ids)
    • 【功能说明】批量计算未读数，用于患者列表汇总。
    • 【使用方法】get_unread_counts(user, conversation_ids)
    • 【参数说明】user: CustomUser, conversation_ids: Iterable[int]
    • 【返回值说明】Dict[int, int]

列表与汇总

  • list_patient_conversation_summaries(studio, viewer)
    • 【功能说明】返回工作室患者会话摘要（患者信息、最后消息、未读数）。
    • 【使用方法】list_patient_conversation_summaries(studio, viewer)
    • 【参数说明】studio: DoctorStudio, viewer: CustomUser
    • 【返回值说明】List[dict]

转移与历史

  • transfer_patient_to_studio(patient, target_studio, operator, reason=None)
    • 【功能说明】原子化转移患者归属，更新历史记录与会话归属。
    • 【使用方法】transfer_patient_to_studio(patient, target_studio, operator, reason)
    • 【参数说明】patient: PatientProfile, target_studio: DoctorStudio, operator: CustomUser, reason: str | None
    • 【返回值说明】PatientStudioAssignment

关键流程

  • 患者沟通：患者会话固定一条，患者侧统一显示“主任工作室”
  • 升级主任：平台医生将问题转发至内部会话，主任仅在内部会话回复指示
  • 主任只读：对患者会话只读，内部会话可写
  • 转移工作室：更新 PatientStudioAssignment，会话仍唯一；历史消息保留原工作室名

附件策略

  • 仅图片
  • 无审计
  • 存储到 media/ 下专用目录，消息记录保存图片地址

里程碑

  1. 完成 chat app 领域模型与基础服务
  2. 患者端 API + 页面联调（轮询、发送、已读、图片）
  3. 医生端面板接入（患者列表未读、内部会话、升级）
  4. 转移工作室流程与历史展示策略完善
